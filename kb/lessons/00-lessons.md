# Lessons ledger — the compounding layer

`created 2026-07-06` · append-only · owner: kb-distiller agent (`.claude/agents/kb-distiller.md`)

**Why this file exists.** CLAUDE.md's rule is *invariants over memory* — but between "learned
it the hard way" and "it's a CI assertion" there is a gap where knowledge evaporates between
stateless cloud runs. This ledger is that gap made visible. Every hard-won lesson gets a row
the moment it's learned; its **enforcement** column tracks how far along the gradient it has
moved (`UNENFORCED` → `protocol` → `test` → `invariant`). `UNENFORCED` rows are the
kb-distiller's standing work queue. Some lessons are genuinely not assertable (venue
behavior, methodology judgment) — those record *why* and stay `protocol`/`ledger-only`;
that's an honest terminal state.

Rules: never edit or delete a row; supersede with a new entry citing the old ID. Every
lesson cites the finding/log entry that produced it (no claim without provenance).
